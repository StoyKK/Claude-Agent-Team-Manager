**Research Pitfall 4** identified that parsers have `@throws {ParseError}` in JSDoc but never actually throw `ParseError`. The `readTextFile` call on line 18 of both parsers rejects with a plain Tauri error. Fix this by wrapping the entire parse body in try-catch.

**File 1: `src/services/agent-parser.ts`**

Wrap lines 18-56 (from `const raw = ...` to the return statement) in a try-catch block. In the catch block:
- If the caught error is already a `ParseError`, re-throw it
- Otherwise, throw `new ParseError("Failed to parse agent file: " + filePath, err)`

The result should be:

```typescript
export async function parseAgentFile(
  filePath: string,
  content?: string
): Promise<AuiNode> {
  let raw: string;
  try {
    raw = content ?? (await readTextFile(filePath));
  } catch (err) {
    throw new ParseError("Failed to read agent file: " + filePath, err);
  }

  let parsed: matter.GrayMatterFile<string>;
  try {
    parsed = matter(raw);
  } catch (err) {
    throw new ParseError("Failed to parse frontmatter: " + filePath, err);
  }

  // ... rest of function unchanged (Zod validation, node construction) ...
}
```

Two separate try-catch blocks: one for `readTextFile` (file access errors), one for `matter()` (malformed YAML). This lets tests distinguish "file not found" from "bad YAML" while both produce `ParseError`.
